# Employee Management Task Plan

目标：把 Notion「员工管理」需求拆成多个可独立验证的前端模块。每个模块由 Codex 在独立分支完成，避免所有改动堆在 `main`。

## 执行方式

在 GitHub Actions 页面运行 `Local Codex Automation`：

```text
https://github.com/zxygzhong-art/nexus-pro-be-coding/actions/workflows/local-codex.yml
```

每次填写：

```text
branch: 下方任务指定的分支名
task: 下方任务内容
```

也可以在 issue 或 PR 评论：

```text
/codex branch=feature/employee-list-filters 任务内容
```

## 任务顺序

### M1: 列表筛选、搜索、统计

分支：

```text
feature/employee-list-filters
```

任务：

```text
基于 Notion 需求「员工管理」做第一阶段可验证前端闭环，范围控制在静态前端原型内。

需求：
1. 员工列表的姓名 / Email / 员工编号搜索要真实过滤列表。
2. 部门、状态、员工类别筛选要真实生效，并且可组合使用。
3. 员工状态枚举统一为：在職、試用中、留停、待加入、離職；不要继续使用不一致的「已停用」。
4. Dashboard 统计要基于当前员工数据计算，至少包含总员工数、在职、试用中、本月离职等现有页面已有指标。
5. 页面加载后员工列表正常显示，筛选和搜索能立即刷新列表和统计。

不做：
- 不新增后端服务、数据库、API、登录鉴权。
- 不做新增编辑保存、批量删除、CSV 导入导出。
- 不重写无关页面。

验收：
- 搜索、部门筛选、状态筛选、类别筛选均能改变列表结果。
- 多个筛选条件组合后结果正确。
- 统计随筛选/数据变化刷新。
- 运行最小必要验证；如果没有自动化测试，至少做 JS 语法检查并说明人工验证路径。
```

### M2: 新增和编辑写回前端数据

分支：

```text
feature/employee-crud-local-state
```

任务：

```text
在 M1 基础上补齐员工新增和编辑的前端数据闭环。

需求：
1. 新增员工保存后写入当前前端员工数据，并刷新列表、筛选结果和 Dashboard 统计。
2. 编辑员工保存后更新当前前端员工数据，并刷新列表、筛选结果和 Dashboard 统计。
3. 保存前做最小字段校验，至少包含姓名、Email、员工编号、部门、状态。
4. 员工编号和 Email 避免重复；重复时给出清晰错误提示。
5. 保存后关闭 modal 或回到合理状态，并给出成功提示。

不做：
- 不接后端 API。
- 不做完整六分页资料。
- 不做导入导出。

验收：
- 新增员工后可以在列表中搜索到。
- 编辑员工状态/部门后，筛选和统计立即反映变化。
- 重复员工编号或 Email 会阻止保存。
```

### M3: 批量删除和 CSV 导出

分支：

```text
feature/employee-bulk-delete-export
```

任务：

```text
在 M1/M2 基础上补齐批量删除和 CSV 导出。

需求：
1. 多选、全选、跨筛选后的选择状态要稳定。
2. 批量删除确认后真实从当前前端员工数据移除。
3. 删除后刷新列表、筛选结果、选择状态和 Dashboard 统计。
4. CSV 导出要基于当前筛选后的员工数据生成真实 CSV 下载。
5. CSV 包含列表中关键字段：员工编号、姓名、Email、部门、职称、状态、类别、入职日、离职日。

不做：
- 不接后端 API。
- 不做审计日志。
- 不做 CSV 导入。

验收：
- 批量删除后被删除员工不再出现在搜索结果中。
- CSV 导出行数等于当前筛选结果数量。
```

### M4: CSV 导入预览和确认导入

分支：

```text
feature/employee-csv-import-preview
```

任务：

```text
把员工导入从 mock 改成真实 CSV 前端解析、预览和确认导入。

需求：
1. 模板下载生成真实 CSV 模板。
2. 上传 CSV 后解析内容，展示预览行。
3. 校验必填字段、状态枚举、员工编号重复、Email 重复。
4. 错误行要显示行号和错误原因。
5. 只有无阻断错误的数据才能确认导入。
6. 确认导入后写入当前前端员工数据，并刷新列表和统计。

不做：
- 不做 XLSX 完整兼容。
- 不接后端 API。

验收：
- 正确 CSV 可以导入并出现在列表。
- 错误 CSV 可以看到错误行和原因。
```

### M5: 员工详情六分页资料

分支：

```text
feature/employee-detail-tabs
```

任务：

```text
补齐员工详情页六分页资料的前端展示和保存闭环。

需求：
1. 六个分页：基本资料、在职资料、学历兵役、通讯资料、保险资料、内部经历。
2. 每个分页读取当前员工数据展示。
3. 编辑后保存回当前前端数据。
4. 条件显示：本国籍/外国籍、留停/离职、毕业/肄业。
5. 列表进入详情、详情返回列表的状态要稳定。

不做：
- 不做后端 API。
- 不做头像真实上传与裁切。
- 不做复杂异动审计。

验收：
- 切换员工详情不串数据。
- 修改详情字段保存后再次打开仍能看到修改结果。
```

### M6: 组织架构联动

分支：

```text
feature/employee-org-linkage
```

任务：

```text
补齐员工管理与组织架构数据的前端联动。

需求：
1. 部门筛选选项来自现有组织架构/员工数据，不再硬编码不一致选项。
2. 新增/编辑员工中的部门、职称、上级、代理人选项来自当前组织和在职员工数据。
3. 上级和代理人不能选择当前员工自己。
4. 员工状态变化后，上级/代理人候选和统计合理刷新。

不做：
- 不接后端组织 API。
- 不做组织架构编辑器。

验收：
- 部门、职称、上级、代理人选项与当前数据一致。
- 编辑员工后相关候选项能同步更新。
```

### M7: 回归验证和体验收口

分支：

```text
feature/employee-management-regression
```

任务：

```text
对 M1-M6 合并后的员工管理功能做回归验证和体验收口。

需求：
1. 检查员工列表、详情、CRUD、批量删除、CSV 导入导出主流程。
2. 修复明显 UI 破版、移动端布局问题和空状态问题。
3. 补齐必要的本地验证说明。
4. 删除调试输出和无用 mock 分支。

验收：
- 主流程可按人工验证路径完整走通。
- 没有明显控制台错误。
- 静态前端服务可正常打开。
```
